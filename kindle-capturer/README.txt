Kindle Auto Capturer v5.29.0 Ultimate
========================================

Kindle Cloud Reader（read.amazon.co.jp）のページを自動でキャプチャし、
PDF / 画像として保存するChrome拡張機能です。

────────────────────────────────────────
■ インストール手順
────────────────────────────────────────
1. Chromeで chrome://extensions を開く
2. 右上の「デベロッパーモード」をONにする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. このフォルダ（kindle-capturer-v5.2.4-ultimate）を選択

※ jspdf.min.js は同梱済みです。追加ダウンロードは不要です。

────────────────────────────────────────
■ 使い方
────────────────────────────────────────
1. Chromeで https://read.amazon.co.jp を開き、対象の本を表示する
2. 読みやすさのため、Kindle設定を「文字サイズ 5 / 行間 中 / 余白 狭」に、
   ブラウザ幅を細長く（A4縦の比率）に調整する
3. 拡張機能アイコンをクリックしてポップアップを開く
4. 「方向」（右開き/左開き）、「本のタイプ」、枚数、間隔（秒）を設定
5.「撮影開始」を押す → 10秒カウントダウン後に自動撮影が始まる
   ※ 撮影中は Kindleのウィンドウ／タブを最前面に保つこと
      （最前面でないと自動的に一時停止し、戻すと再開します）
6. 撮影完了後、編集画面（viewer）が自動で開く
   ※ 開いた時点で「余白カット」が自動適用され、本文だけに切り抜かれます
7. 不要ページを削除・調整し、「保存」→ タイトル/著者を確認して PDF 出力

────────────────────────────────────────
■ 余白カット（自動クロップ）
────────────────────────────────────────
・編集画面を開くと、各ページの周囲の余白（ブラウザの白地・リーダー背景）を
  自動検出して本文だけに切り抜きます。
・左右の黒帯（レターボックス）はキャプチャの副産物のため、
  余白カットのON/OFFに関係なく常に除去されます。
  OFF = 黒帯だけ除去してページの余白はそのまま残す
  ON  = 黒帯に加えて白余白もカットして本文だけに切り詰める
・黒帯を除いた結果が縦長になれば自動的に「単ページ」として扱います
  （表紙や単ページが黒半分で割れてしまう問題への対策）。
・【文章タイプのみ】図版やグラフが分割境界を跨いでいるページは、自動的に
  分割せず1ページに収めます（図版が真っ二つになりません）。
  グラフの軸線や図版の枠線のような「境界を横切る水平線」も検出するため、
  バーの隙間などに境界が落ちて図が切れることもありません。
・★v5.21.0【マンガ・雑誌（固定レイアウト）】は「レイアウトは全編一定」を
  徹底: 見開きページは必ず左右に分割し（見開きイラストも紙の本と同様に
  左右へ割ります）、分割位置は全編統一のノド位置に固定します。
  ページごとにカット位置が変わることはありません。
  ※ 見開きイラストを1枚のまま残したいページは「結合」ボタンで戻せます。
・★v5.22.0 ノド位置検出の改善と安全装置:
  - 白地の雑誌はノドに明暗差が無く、誌面デザインの色境界（毎ページ同じ
    位置）をノドと誤検出して左右ページの幅が不揃いになることがありました。
    「ノド近傍のほぼ白の縦の継ぎ目」の全ページ投票を最優先に変更。
  - さらに固定レイアウトでは、左右ページは同幅のはずなので、中央から
    ±4%を超えるノドは誤検出とみなして中央で分割します。
  - 1P表示の表紙が見開きサイズの枠の中央に写っている場合（左右に大きな
    白帯）、分割せず内容中心に半ページ幅の枠を当てます。
・★v5.22.0 編集画面を開いたとき、撮影時にすり抜けた連続重複ページを
  自動で削除します（件数を左上に表示。Ctrl+Zで取消可能）。
・★v5.23.1/v5.24.0 表紙（先頭キャプチャ）と裏表紙（末尾キャプチャ）は
  マンガ・雑誌でも「全ページ同サイズ」の枠統一を使わず独立判定し、
  内容の外接矩形そのものでカットします。表紙・裏表紙はリーダーの
  表示レイアウトが本文と異なることが多く、統一枠では白帯が残るためです
  （内容が横長＝見開き表紙の場合は通常の分割処理になります）。
・★v5.24.0 左右ページの幅を完全に同一化: ノドが厳密な中央からわずかに
  ズレている本でも、境界から遠い側に合わせて短い側を白マージン方向へ
  広げ、偶数ページと奇数ページのカットサイズを必ず揃えます。
・★v5.25.1 表紙・裏表紙の内容検出を強化: 背景の明るさを自動判別し、
  暗い背景（ダークテーマのリーダー等）でも表紙の縁を検出できるように。
  判定結果はコンソール（F12）に「[表紙/裏表紙判定]」として記録されるので、
  効いていない場合の原因特定に使えます。
・★v5.26.0 撮影データの保存方式を刷新（前の本が表示される問題の修正）:
  - 従来は全ページを1回で保存しており、ページ数の多い本でサイズ超過で
    静かに失敗すると、前回の本のデータが残ったまま編集画面が開いて
    「前に撮った本が表示される」事故が起きていました。
  - 画像を数枚ずつ分割して保存し、最後にメタ情報を書き込む方式に変更。
    保存に失敗した場合はバッジ「ERR」と赤い警告バナーで明示します。
  - 1枚も撮影できなかった場合（本の終端で開始した等）もバッジ「0枚」と
    警告バナーで通知します（編集画面は開きません）。
  - 編集画面の左上に『書名』と撮影日時を表示するので、どの本のデータを
    見ているかが常に確認できます。
・★v5.26.1 更新忘れの自動検知: ファイルを更新した後に chrome://extensions の
  「⟳ 更新」を押し忘れて拡張本体（撮影側）が旧バージョンのまま動いている
  場合、編集画面の上部に赤い警告バーを表示します。この警告が出たら
  ⟳更新→撮影からやり直してください（旧版の撮影修正は反映されていません）。
・★v5.27.0 撮影中の進捗表示: Kindle画面の右下に
  「📷 33枚｜33/136（残り約103）」のような進捗ピルを表示します。
  ページ数はリーダー自身のページ表記（シークバー等）から読み取るため、
  あと何ページくらいで終わるかがひと目でわかります
  （リフロー本では「位置No.」基準の数字になります。ピルは撮影の
  瞬間だけ自動で透明化されるので、キャプチャには写り込みません）。
・★v5.26.2 「撮影は終わったのに編集画面が開かない」の修正:
  - タイトル・著者の自動取得でエラーが起きると、保存も編集画面も実行されず
    黙って終わることがありました。取得失敗でも保存を続行するように修正。
  - 撮影後の処理全体を保護し、何が起きても必ずバッジ（ERR/0枚）と
    赤いバナーで結果を表示します。エラーバッジは消えずに残ります。
  - 保存後に読み戻し検証を行い、本当に保存できたことを確認してから
    編集画面を開きます。
・★v5.27.1 保存エラーの詳細を記録・表示:
  - 保存に失敗した場合、「どの段階で・何のエラーか・ストレージ使用量」を
    記録し、赤バナー（60秒表示）とバッジ「ERR」に加えて、次に編集画面を
    開いたときにも上部の赤いバーでエラー詳細を表示します。
  - バナーを見逃しても、編集画面を開けば失敗の理由が必ず確認できます
    （次の撮影が成功すると記録は消えます）。
・★v5.28.0 ページ数の多い本で「撮影後に何も起きない」問題の根本修正:
  - 従来は撮影した全ページをメモリに溜めてから最後に保存していたため、
    ページ数の多い本（200ページ超）では拡張の内部プロセスがメモリ不足で
    落ち、保存も通知も行われないまま終わることがありました。
  - 撮影しながら1枚ずつ即ストレージへ保存する方式に変更。メモリ使用量は
    ページ数に関係なく一定になり、何百ページの本でも安定します。
  - 前回の本のデータは「新しい撮影の1枚目が保存できた時点」で削除します
    （撮影が1枚もできなかった場合、前回のデータは無傷で残ります）。
・★v5.28.1 はみ出しガードを「猶予付き」に変更（撮影が始まらない問題の修正）:
  - リーダーが表紙などを画面より大きく描画する本では、はみ出しが
    ウィンドウ調整では直せず、ガードが永久に待ち続けて撮影が
    始まらない・進まないことがありました。
  - はみ出し検知時は約10秒待ち、直らなければ「端が切れる可能性」を
    警告した上で撮影を続行します（開始カウントダウン・各ページ共通）。
    端が切れたページは後から「差し替え」で修復できます。
・★v5.28.2 開き方向（右開き/左開き）の自動修正:
  - 方向設定が本と逆だと「次のページ」の代わりに「前のページ」を
    送ってしまい、表紙から進めず終端と誤判定して止まっていました。
  - 撮影の序盤（1〜2枚目）でページが進まない場合は設定ミスとみなし、
    反対方向へ自動で切り替えて続行します（画面に通知を表示）。
    本の終端では切り替えないので、逆走して戻り撮影する心配はありません。
  - 事前に方向を正しく選ぶのがベストですが、間違えても撮影は完走します。
・★v5.28.3 「0枚」終了の理由を明示:
  - キャプチャAPIの連続失敗や1枚目の保存エラーで0枚に終わった場合、
    従来は汎用メッセージがエラー詳細を上書きして原因が見えませんでした。
  - 失敗理由（キャプチャ/画像書き込み等＋エラー内容）を「0枚」バナーに
    そのまま表示し、編集画面の赤バーにも残るようにしました。
・★v5.29.0 文章（リフロー）の本のページサイズ不揃いを修正:
  - 原因: リーダーUI（上端の書名・下端のページ番号/「読書速度を学習中…」）が
    キャプチャに写り込み、ページ内容として検出されていました。特に左下の
    学習中テキストは出たり消えたりするため、枠の幅がページごとにバラバラに。
    さらに下端中央のページ番号が2段組の列間（分割の谷）に載り、見開き分割が
    「図版が跨いでいる」と誤ブロックされて横長のまま残るページもありました。
  - 対策①（撮影時）: ページ送りボタンと同様に、リーダーUIも撮影の瞬間だけ
    透明化します。画面の上下端にある小さな固定要素を位置ベースで特定するため、
    リーダーの画面更新でクラス名が変わっても機能します。
  - 対策②（編集画面）: 撮影済みデータ向けの救済。文章タイプでは、本文から
    離れた上下端の薄いテキスト帯（書名/ページ番号）を検出枠から除外します。
    再撮影しなくても、編集画面を開き直すだけでサイズが揃います。
    除外の実施状況はコンソールに [リーダーUI除外] として記録されます。
・マンガ・雑誌（固定レイアウト）は「ページ枠は全編で一定」という前提で、
  全キャプチャの検出結果の中央値でページ枠を統一します（全ページ同サイズ）。
  ページ内容の黒縁（黒コマや黒背景の表紙）で帯検出がぶれたページも、
  多数派の帯位置をアンカーにして同じ枠に揃えます。
・本文が見開き表示でも、表紙・裏表紙は1ページ表示で撮れることがあります。
  各キャプチャを縦横比で「見開き / 1ページ」に自動分類し、1ページ表示の
  表紙・裏表紙には見開きの半分幅の枠を適用します（分割されません）。
・撮影開始時、画面が安定する（表紙などの描画が終わる）まで待ってから
  1枚目を撮ります（バッジに「...」表示。最大約8秒）。
・見開きの分割位置（ノド）も全キャプチャの検出の中央値で統一します。
  隣ページの切れ端が写り込まず、左右ページの幅も全編で揃います。
・上部の「余白カット:ON/OFF」ボタンでいつでも切り替え可能（削除や並びは保持）。
・全面が写真等で余白が無いページは、そのまま（切り抜かれません）。
・うまく切り抜けない場合は OFF にして元のまま保存できます。
・切り抜き後は文字が見切れないよう、各辺に少し余白を残します
  （白背景側は広め、黒帯側は黒縁が写り込まないよう最小限）。
・ほぼ白紙で検出が不安定なページは、近くの正常なページと同じ位置・サイズに
  自動で揃えます。インクが十分あるページは、サイズが他と違っても
  そのページ固有のレイアウト（縮小表示の表紙等）として尊重します。

■ マージン調整
────────────────────────────────────────
ツールバーの「マージン」メニューで、本文の外側の白マージンを
「なし / 狭い / 標準 / 広い」から選べます。
・ページを選択（クリック、Shift+クリックで範囲、チェックボックス）して
  選ぶと、その選択ページだけに適用されます。
・何も選択していない状態で選ぶと、全ページに適用されます。
・未指定のページは従来どおり: 文章タイプ＋余白カットONのとき「標準」相当。
※ 選択の解除は、チェック済みのカードをもう一度クリック（または x キー）。

────────────────────────────────────────
■ ページの差し替え・追加（一部だけ撮り直したいとき）
────────────────────────────────────────
表紙や数ページだけ失敗したときに、その部分だけ手動の画像で差し替え・追加
できます（全体の撮り直しは不要）。
・差し替え: 対象ページをクリックで選択 →「差し替え」ボタン → 画像ファイルを選ぶ
            → そのページが選んだ画像に置き換わります。
・追加    : 基準ページを選択 →「追加」ボタン → 画像ファイル（複数可）を選ぶ
            → 選んだページの後ろに挿入されます。
・差し替え/追加した画像は自動クロップ・分割の対象外で、そのまま1枚として扱います
  （周囲に余計な部分があればダブルクリックのトリミング編集で切り抜けます）。
・変更は元データに保存されるので、編集画面を再読み込みしても維持されます。
・用意する画像: macなら Shift+Cmd+4 でページ部分だけスクリーンショットを撮るのが簡単です。

────────────────────────────────────────
■ 見開きの分割・結合（見開き検知 / 2分割 / 結合）
────────────────────────────────────────
自動分割が見送って見開きのまま残ったページを修正する3つのボタンです。

●見開き検知（★v5.19.0・一括処理）
・全ページから「横長のまま残っているページ」を自動検知し、件数を
  確認してから一括で2分割します。1ページずつ探して選ぶ必要はありません。
・本当の見開きイラストも分割されるので、分割しすぎたページは
  「結合」で戻してください。

●2分割（★v5.18.0・個別処理）
・分割したいページを選択（複数可。無ければクリックでフォーカスだけでもOK）
  →「2分割」ボタン。
・分割位置は「ノド付近の白い継ぎ目」を自動で探してそこで切ります
  （見つからなければ全編統一のノド位置、それも無ければ中央）。
・図版保護は行いません（明示的な指示のため必ず分割します）。
・右開き/左開きの設定に従ってページ順が決まります。

●結合（★v5.19.0・分割の取り消し）
・誤って2分割したページや、自動分割を1枚に戻したいページを選択
  →「結合」ボタン。同じ撮影画像から作られた隣のページと1枚に戻ります。
・左右どちらか片方を選ぶだけでOK（隣は自動で見つけます）。
・手動2分割だけでなく、自動分割された見開きの取り消しにも使えます。

※ 分割・結合の結果は保存前の編集画面上のみ有効です（再読み込みでリセット）。

■ トリミング編集（余白カットの手動調整）
────────────────────────────────────────
自動の余白カットが上手くいかないページ（黒背景のページ等）は、
手動でカット枠を調整できます。
・ツールバーの「トリミング」ボタン、またはページカードのダブルクリックで
  トリミング編集が開きます。
・枠の内側ドラッグで移動、端・角のハンドルをドラッグでサイズ変更。
・★v5.20.0 赤いガイド線を表示:
  - 赤の破線 = 自動検出した本の縁（元のカット枠）。枠をどれだけ
    動かしたかの目安になります。
  - 赤の実線 = 現在の枠の縦横の中心線。ノドが中央に来ているか、
    左右の余白が対称かの確認に使えます。
・「適用」で確定（手動トリミングは余白カットON/OFFより優先されます）。
・★v5.17.0「選択ページにも適用」: このページで動かした各辺の調整量を、
  選択中の全ページの枠にも反映します（各ページ自身の枠を基準に同じだけ
  辺を動かすので、見開きの左右ページが混ざっていてもズレません）。
  使い方: 直したいページを選択（Shift+クリックで範囲選択可）
  → 「トリミング」→ 枠を調整 → 「選択ページにも適用」。
・「自動に戻す」で自動切り抜きに復帰。Escでキャンセル。

────────────────────────────────────────
■ 本のタイプ（重要）
────────────────────────────────────────
Kindleの本はタイプによって最適な処理が違うため、タイプ別に切り替えます。

・自動    : 撮影時にページ構造から自動判定します（迷ったらこれ）。
            リフロー本（「位置No.」表示）→ 文章、大きな画像描画 → 固定レイアウト。
            固定レイアウトはさらに本文中盤の色を調べ、白黒→マンガ / カラー→雑誌。
・文章    : 小説・ビジネス書など文字中心（リフロー型）。余白カットON
            （文字が見切れない保護マージン付き）。
            横長のウィンドウで撮影した場合は、表紙から求めた「本の縦横比」に
            合わせて自動でページ分割します。分割位置は「最も幅の広い空白帯」
            （見開きのノドや行間）にスナップするため、ノド際の章タイトルや
            図版が切れることはありません（縦書きは右→左の順）。
            出力には外側に白マージンを自動で付けるので、文字が枠ギリギリに
            ならず端末で読みやすい縦長のPDFに仕上がります。
・マンガ  : 固定レイアウト・白黒。黒帯除去＋見開き自動分割＋余白カットON。
・雑誌    : 固定レイアウト・カラー。帯除去のみで誌面の余白は残す（余白カットOFF）。

判定結果は編集画面の左上「合計: N ページ（タイプ: 〇〇）」に表示されます。
※ 編集画面の「分割」セレクタと「余白カット」ボタンでいつでも上書きできます。
  切り替えると全ページを再構成します（削除やUndo履歴はリセット）。
  表紙やページが左右に割れてしまった場合は「分割:なし」に切り替えてください。
  撮影し直しは不要です。

※ 以前のバージョンは常に左右分割していたため、単ページ表示の本が
   半分に割れてしまう不具合がありました。v5.2.4 で修正済みです。

■ 完了検知（自動停止）
────────────────────────────────────────
・本の最後までいくとページ送りが効かず、同じページを撮り続けてしまいます。
・そのため、直前と同じページが3回連続で検出されたら「終端」とみなして
  自動的に撮影を停止します（拡張アイコンのバッジに一時的に「END?」と表示）。
・連続して検出された重複フレームは保存されません（1枚だけ残します）。

■ 重複削除（編集画面）
────────────────────────────────────────
・上部の「重複削除」ボタンで、連続する同一ページを1枚だけ残して削除します。
・完了検知をすり抜けた重複や、すでに撮影済みのデータの掃除に使えます。
・削除は Ctrl+Z で取り消せます。

■ トラブルシューティング
────────────────────────────────────────
・撮影が始まらない
  → read.amazon.co.jp のタブがアクティブか確認。別サイトでは開始できません。
・途中で止まる／ページが送られない／1ページだけで終わる
  → Kindleウィンドウを最前面に保つ。間隔（秒）を長め（5〜6秒）にする。
  → 1ページだけで終わる場合はページ送りが効いていません。
    v5.11.2: iframe描画のリーダー（コミック等）にもキー送出。
    v5.16.2: 新リーダーの矢印ボタン(kr-chevron)に対応し、クリックを
    pointer/mouse の一連イベントで送出。リーダー本体を描画している
    フレームへのタップゾーン送出も追加。
  → ★重要: 拡張のファイルを更新した後は chrome://extensions で必ず
    「更新（⟳）」を押すこと。押さないと旧バージョンのまま動きます
    （タブ名の「Kindle Auto Capturer vX.X.X」が新版か確認できます）。
  → Amazon側のUI変更でページ送りボタンが変わった場合は動作しないことがあります。
・文字がページの端で欠ける／見切れる／表紙が左右に分かれて撮れる
  → 元のキャプチャの時点でページがウィンドウからはみ出して、左右の文字が
    失われています（後処理では復元できません。切れたキャプチャに余白を
    足しても直りません）。
    ★v5.13.0の「はみ出し防止ガード」：
      文章・マンガ・雑誌すべてのタイプで、ページが画面からはみ出して
      いる間は撮影しません。撮影前・および各ページ撮影の直前に自動で
      チェックし、はみ出し中は画面に黄色い警告を出して待機します
      （バッジ「はみ出」）。ウィンドウ幅を広げる／文字サイズを小さく
      して全体が見えるようにすると自動で再開します。
    ★v5.16.3: ページ全体が見えているのに「はみ出しています」警告が
      出続けて撮影が進まない誤検知を修正（コミック等のリーダーが
      前後ページを画面外に先読みする構造に反応していたもの。
      固定レイアウト本は「見えているページ画像自体の位置」だけで
      判定するように変更）。
    → 対処: 文章の本は文字サイズを小さめ＋余白「狭」に。
      固定レイアウト本はウィンドウを「ページ全体が見える幅」に。
    → すでに切れて撮れてしまったデータは復元できないため、撮り直しが必要です。
・ページ送りの矢印（›）が画像に写り込む
  → 撮影時に自動で透明化しています。それでも残る場合はKindleのUI仕様変更が
    原因です（background.js の hideCSS のセレクタ調整が必要）。
・PDFが真っ二つ / 左右にズレる
  →「ページ構成」を「単ページ」に切り替える。
・PDFが保存できない
  → 拡張を再読み込み（chrome://extensions で更新ボタン）してから再試行。
