## 製造業

技能伝承、安全衛生教育、品質管理、グローバル展開の観点で動画とインタラクティブ学習の親和性が極めて高い業界。日本の製造業は工場現場の技能伝承課題が深刻で、ベテラン技能者の暗黙知を動画+インタラクションで形式知化するニーズが高まっている。

### よくある課題

- 熟練工の退職に伴う技能伝承で、動画に撮るだけでは若手の理解が定着しない
- グローバル拠点の作業員に対する多言語の作業手順研修・安全教育の運用負荷が大きい
- ISO 9001、ISO 14001、IATF 16949などの認証維持に必要な教育記録の自動化が進んでいない
- 新人配属時のオンボーディング動画が長尺で集中できない
- 製品マニュアル動画があっても顧客が必要箇所にたどり着けず、サポート問い合わせが減らない

### 活用例

- **技能伝承動画への暗黙知層の追加**：要所にホットスポット・確認問題を挿入し「なぜここで力加減を変えるのか」「次に何を確認するのか」を明示化。若手のつまずきポイントを可視化しコンテンツを継続改善
- **安全衛生教育**：年次安全教育・KY（危険予知）訓練に判断問題・選択分岐。労働安全衛生法に基づく教育実施記録・修了確認エビデンスを自動生成
- **グローバル拠点向け作業手順研修**：多言語対応（字幕・選択肢翻訳）+AIアシスタントの自動翻訳で本社作成の手順動画を海外拠点に展開。拠点ごとの理解度・進捗を本社LRSで一元把握
- **製品マニュアル・サポート動画のセルフサービス化**：章立てとホットスポットで症状・型番から該当箇所に直接ジャンプ。一次解決率向上に直結

### 導入事例

- **Würth（ウルト）グループ（ドイツ）**：自動車補修・整備市場の世界トップ企業（売上1.4兆円、従業員約6万3,000人、全世界300万社以上の顧客）。「車種と場面に応じて誰でも自動車用充電器を選べる」インタラクティブ動画をhihahoで制作し、業界権威あるPlatinum Viddy賞（米国の映像制作優秀作品賞）を受賞。製造業への安心材料・モデル事例として強力
- **火災現場での緊急対応マニュアル（海外消防組織）**：ストレスがかかる緊急時の火災訓練を現場再現訓練の代替としてhihaho化。「緊急事態での判断力が向上した」隊員90%、「事前知識や準備の必要性を感じた」隊員100%。消防・警察・医療・防衛・原発・化学プラントなど緊急対応訓練を行う全組織に応用可能

### 提案のコツ

- 品質管理部門・人事部門・サポート部門の3部門に同時アプローチできるテーマ設定で社内合意形成が早い
- グローバル展開視野の場合、データ保管リージョン・GDPR適用範囲の確認を商談初期に実施
- 既存の社内ポータル・eラーニングシステムへの組み込み（SSO、LTI、SCORM）を必ず確認
- 「技能伝承」を切り口にすると人手不足課題と連動して経営層の関心が高い

## IT・SaaS

プロダクトトレーニング、新人エンジニア研修、カスタマーオンボーディング、開発者向けドキュメント補完など活用場面が多い。自社サービスにhihahoを組み込んでエンドユーザー向けに提供するケースもある。

### よくある課題

- 自社SaaSの操作研修動画を顧客が完了せずチャーンにつながる
- 新人エンジニア研修で技術スタック・社内ツール・コーディング規約を自走的に習得できる仕組みが弱い
- CS部門のオンボーディングが属人的で品質均質化が難しい
- セールスエンジニアのデモ動画で視聴後の関心度が分からず追客優先順位が立てづらい
- プロダクトアップデート動画を顧客が見ないため新機能の活用率が上がらない

### 活用例

- **SaaSプロダクトの顧客オンボーディング**：初期設定動画にステップごとのチェック問題・ロール選択分岐でTTV（Time to Value）改善。視聴データはCRMに連携
- **エンジニア向け技術研修**：勉強会・新人研修・コーディング規約解説にコード問題・選択肢・参考リンク。社内技術知識ベースとして長期蓄積
- **セールスエンジニア向けデモライブラリ**：デモ動画にホットスポット+「気になる機能を選んでください」分岐。MA連携でリードスコアリングに統合
- **プロダクトアップデート動画**：「使う予定がある機能はどれか」等の調査クイズでプロダクトチームへの定量フィードバック

### 導入事例：通信会社 — 問合せ数134%成長

国内大手通信事業者（@nifty光）がサービス紹介動画にhihahoを導入した結果、前年比問合せ数134%成長。動画視聴中のCTA・選択分岐・関心領域プロファイリングにより、ブランディング動画から「リード獲得装置」へ転換した事例。リード獲得KPI改善を求めるBtoCサービス事業者への強い数値根拠。

### 提案のコツ

- 自社プロダクトを顧客にどう体験してもらうかという視点で、CS・セールス・マーケ・プロダクトの4部門にまたがる議論を促せる
- 自社サービスにhihahoを組み込むOEM・パートナーシップ型の利用パターンも提案視野に
- 3種類のAPI（API to create or upload / Data API / JavaScript API）・Webhook対応の詳細仕様を技術担当者向けに別途資料化しておくと技術評価がスムーズ。自社サービス組み込みはJavaScript API、データ連携はData API、業務自動化はAPI to create or uploadが鍵。公式APIドキュメント: https://api-docs.hihaho.com/
