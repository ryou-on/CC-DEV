# ヨルマチ（仮称）企画書 — ナイトタイムエコノミー向けエンタメマップ

> Phase 0 企画・要件定義 / 作成日: 2026-06-10 / v0.1
> MVP: https://cc-dev-ps7.web.app/yorumachi/

---

## 1. コンセプト（1行）

**「今から夜遊びたい」と思った瞬間に、現在地から徒歩圏で"今夜やってる"小箱が見つかるマップ。**

---

## 2. 課題

- 小規模ライブハウス・バー・小劇場・スナックの「今夜の情報」はまとめサイトに載らない
- 載っていても「現在地からの距離」「いま営業しているか」が分からない
- 店側はX/Instagramで告知するが、フォロワー外には届かない（新規客に発見されない）
- 長期滞在の外国人観光客は「観光地ではない日常の夜遊び」を探す手段がない

## 3. ソリューション

GPS連動マップで以下を一画面に統合:

| 機能 | 内容 |
|---|---|
| 今夜フィルタ | 「今日イベントをやっている場所」だけを表示 |
| 営業中フィルタ | 深夜表記（〜29時）対応の営業判定 |
| 徒歩圏リング | 現在地から徒歩5/10/15分の円を地図に描画 |
| ジャンル | ライブ／クラブ・DJ／バー・スナック／お笑い・舞台／深夜メシ |
| 送客導線 | 「行き方」「予約・問合せ」ボタン → 手数料計上ポイント |
| 日英切替 | インバウンド長期滞在者向け |

**テストエリア: 大塚・池袋**（小箱密度が高い／JR・地下鉄結節点／豊島区はナイトカルチャー施策に積極的）

## 4. ターゲット

1. **地元・沿線の夜遊び層**（20〜40代）: 「今夜なにかないかな」を週1回以上思う人
2. **長期滞在インバウンド**: 観光に飽き、日常的な夜の娯楽を探す層（日英対応で取り込む）
3. （供給側）**集客に悩む小箱オーナー**: SNS告知の届かない新規客が欲しい

| サービス | 性質 | 「今夜×現在地」 |

| [LIVEHOUSe+（イープラス）](https://livehouse.eplus.jp/) | 全国1,000店舗の会場情報 | ✕ |
| [livehouse.tv](https://livehouse.tv/) | 出演・レンタル向け情報 | ✕（演者向け） |
| [Ticket Bird](https://apps.apple.com/jp/app/ticket-bird-%E8%A1%8C%E3%81%8D%E3%81%9F%E3%81%84%E3%83%A9%E3%82%A4%E3%83%96%E3%81%8C%E8%A6%8B%E3%81%A4%E3%81%8B%E3%82%8B/id1660900257) | アーティスト起点のライブ発見 | ✕（推し起点） |

**→ 「いま・ここから徒歩圏・今夜」のリアルタイム位置起点ディスカバリーは空白地帯。**
既存勢は「会場DB」「チケット」「推し起点」であり、"目的地未定の夜"を拾うプロダクトがない。

## 6. 最大の課題 = イベントデータ収集（ここが本体）

マップUIはコモディティ。**「今夜のデータが揃い続ける仕組み」が参入障壁になる。**

| 手段 | 内容 | コスト |
|---|---|---|
| ① 店舗セルフ投稿 | LINE公式アカウント or 1分フォームで「今夜の告知」投稿 | 低（継続率が課題） |
| ② **SNS告知のAI構造化** | 店のX/Instagram投稿をClaude APIで解析→日時・料金・出演者を自動抽出してイベントデータ化 | 中（**技術的差別化の核**） |
| ③ コミュニティ投稿 | 互助会等のネットワークで「今夜ここ行く」情報を集める | 低 |

②は表さんの既存スキル（Claude API統合）がそのまま活きる。店側は「いつものSNS告知をするだけ」で掲載される＝営業コストが激減する。

## 10. KPI（Phase 2〜）

- 掲載店舗数（目標: 大塚エリア20軒）
- 「今夜情報」の鮮度（当日イベント掲載率）
- WAU / 金土夜のDAU
- 送客CV（行き方タップ・予約タップ）
- 店舗側の継続投稿率（月次）

## 11. リスク

| リスク | 対策 |
|---|---|
| 二面市場の鶏卵問題 | 供給側から固める。店舗ゼロ営業を避けるためAI抽出（SNS公開情報）で初期データを自動生成し「もう載ってます」営業 |
| データ鮮度が落ちると一気に無価値化 | 当日0時に全イベント自動失効・「情報なし」を正直に表示。鮮度をプロダクトの憲法にする |
| Google Maps の機能拡張 | 小箱×夜×日本語SNS告知のロングテールはGoogleが構造的に拾えない領域。ここに集中 |
| 公開SNS情報の転載に関する店舗感情 | 掲載時に店舗へ通知→ワンタップでオーナー登録 or 削除依頼できる導線を用意 |

## MVP（v0.1.0）実装済み機能

- ダークテーマの夜仕様マップ（Leaflet + CARTO dark / 大塚・池袋）
- GPS現在地 + 徒歩5/10/15分圏リング描画
- 架空デモ店舗14軒（ライブ/クラブ/バー/舞台/深夜メシ）
- 今夜イベント・営業中（深夜29時表記対応）・ジャンルのフィルタ
- 距離順カードカルーセル（徒歩分数表示）⇄ ピン連動
- 送客デモ（予約タップを手数料1件としてカウント、累計と想定売上を表示）
- 日本語/英語切替（インバウンド検証用）
- Googleマップ徒歩経路連携／デバッグログコピー（🐞）
